# DUT v35 — support chat and admin replies

Что изменено:

1. Исправлены выпадающие списки в настройках дизайна.
   - В тёмной теме пункты select теперь имеют нормальный фон и читаемый текст.
   - Проблема, где варианты в списке были почти невидимыми, исправлена через `select option` и `color-scheme`.

2. Добавлен мини-чат поддержки в углу сайта.
   - Чат появляется у авторизованного пользователя.
   - Пользователь может написать администратору прямо с сайта.
   - Переписка сохраняется на backend в `data/db.json`.
   - После выхода и повторного входа пользователь снова видит историю сообщений.

3. Доработаны обращения в админке.
   - В админке видно, от кого пришло обращение: имя, email, тема, дата.
   - Обращение теперь отображается как переписка.
   - Администратор может отправить ответ пользователю.
   - Ответ сразу сохраняется и отображается пользователю в мини-чате.
   - Обращение можно закрыть, вернуть в новые или удалить.

4. Сохранение аккаунтов.
   - Зарегистрированные пользователи сохраняются в `data/db.json`.
   - После регистрации можно выйти и снова войти под тем же email и паролем.
   - Для постоянного хранения на Render после перезапусков лучше подключить Persistent Disk и указать `DATA_DIR`.

Проверка:

```bash
node --check server.js
node --check public/app.js
node --check public/admin/admin.js
npm start
```

Сценарий проверки:

1. Зарегистрировать пользователя.
2. Выйти и снова войти под ним.
3. Открыть мини-чат в углу сайта.
4. Отправить сообщение администратору.
5. Открыть `/admin/`.
6. Перейти в раздел «Обращения».
7. Ответить пользователю.
8. Вернуться в пользовательский аккаунт и открыть мини-чат — ответ должен быть виден.
